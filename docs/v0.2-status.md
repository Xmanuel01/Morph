Morph v0.2 Status vs Morph.spec

Legend:
- Implemented: behavior matches v0.2 scope.
- Partial: works with v0.2 constraints, but missing full spec semantics.
- TODO: not implemented.

1) Modules + use resolution
- File-based module resolution (Implemented) `morphc/src/loader.rs` (ModuleLoader::resolve_module_file).
- use foo.bar => src/foo/bar.morph or src/foo/bar/index.morph (Implemented) `morphc/src/loader.rs`.
- Module loading with dependency scan (Implemented) `morphc/src/loader.rs` (collect_uses, load_module_path).
- Module records exposed at runtime (Implemented) `morphrt/src/runtime.rs` (eval_module_record, define_path).

2) morph run . (project entry)
- morph.toml discovery and src/main.morph entry (Implemented) `morph/src/main.rs` (resolve_entry).
- Package loading from root + entry (Implemented) `morph/src/main.rs` + `morphc/src/loader.rs`.

3) Better diagnostics (line/col + snippet)
- Parse/lex errors include line/col + snippet (Implemented) `morphc/src/parser.rs` (ParseError::fmt, format_snippet).
- CRLF line endings are normalized by `str::lines` for snippet display (Implemented) `morphc/src/parser.rs` (format_snippet).

4) Runtime stack traces
- Stack frames for user + native calls (Implemented) `morphrt/src/runtime.rs` (RuntimeError::add_frame, eval_call).

5) Minimal formatter / formatting validation
- Indentation-based formatter + --check validation (Implemented) `morphc/src/formatter.rs`.
- CLI wiring for morph fmt (Implemented) `morph/src/main.rs`.

6) Policy enforcement MVP
- Default deny for capabilities (Implemented) `morphrt/src/runtime.rs` (check_capability).
- allow/deny rules for io/fs/net/process/model/memory/tool (Implemented) `morphrt/src/runtime.rs` (Policy::is_allowed).
- Capability matching is prefix-based and deny overrides allow (Implemented) `morphrt/src/runtime.rs` (capability_matches, Policy::is_allowed).
- Tool and std stub gating by policy (Implemented) `morphrt/src/runtime.rs` (define_tool_stub, install_builtins, eval_call).
- Model/memory stubs are exposed via `std.ai.*` to avoid keyword field access (Implemented) `morphrt/src/runtime.rs` (install_builtins).

Divergences / v0.2 constraints
- Policy filters are parsed but ignored at runtime (Partial) `morphrt/src/runtime.rs` (register_policy).
- Only a policy named "default" becomes the module policy; no block-level policy scoping (Partial).
- Policies are not intersected; single active policy enforced per scope (Partial).
- Formatter adjusts indentation only and normalizes line endings; it is not a full AST rewriter (Partial).
